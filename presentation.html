從一句想法到可上線的問卷網站

以「Satir 五種自由」雷達圖問卷為例，示範如何透過連續提問，把概念、資料、視覺、權限、後台與上線流程逐步完成。

開啟問卷 GitHub Repo

成果概覽

開發路線

一個專案可以這樣問出來

  1. 先描述想法像問卷網站，答案轉成雷達圖。
  2. 調整量表從 1-10 改成 0-10。
  3. 釐清資料結構同五題、三次回答、三種意義。
  4. 補視覺規則重視紅、使用藍、熟悉黃。
  5. 做第一版看畫面後逐步修正文字、尺寸、排版。
  6. 接後台與上線GAS、Google Sheet、GitHub Pages、QR Code。

你的提問示範

需求不是一次說完，而是逐步變清楚

我想設計一個類似問卷的網站，題目歸類為五大項，回答後轉成雷達圖。
先將每一題由 1~10 調整成由 0~10。
如果同時有 20 人回應，雷達可以同時顯示這 20 人的回應嗎？
我的題組共 5 題，題目都一樣，第一次看重視程度，第二次看應用程度，第三次看熟悉程度。

視覺決策

讓圖表規則先固定，使用者才看得懂

重視程度紅色
使用頻率藍色
熟悉程度黃色
  • 完全重疊時，後兩條線改成虛線，避免其中一條被蓋住。
  • 個人頁保留三線總覽；全部個人頁採姓名 + 2x2 小雷達圖。
  • 14 吋筆電螢幕優先，調整圖卡尺寸與文字距離。

迭代修正

看畫面後，請 AI 修「具體問題」

你指出的問題

  • 文字不見或重疊。
  • 表格欄位擠在一起。
  • 雷達圖太小或文字離圖太近。
  • 全部個人一排太寬，改成 2x2。

好的提問方式

「如圖片所示，某些文字重疊，請讓圖稍微大一點，但仍保持在區塊內，並把文字與圖的距離拉開。」

資料後台

Google Sheet 不是只寫入，還要能讀回

學生掃 QR 自動帶入 GAS URL 填答送出 寫入 Google Sheet 講師載入後台資料

這一步後來修正過：一開始只有寫入 Sheet，講師前台仍看不到後台資料；後來加上 GAS 的讀取入口，講師模式才會把 Sheet 資料載回來。

權限設計

學生入口與講師入口分開

學員端

只看填答與個人結果。學員連結會保留 `student=1`，並自動隱藏講師模式與後台設定。

講師端

用一般網址或 `?admin=1` 進入。輸入講師密碼後，可以看團體平均、全部個人與後台工具。

提醒同事：GitHub Pages 是靜態網站，這是課堂使用的前端保護；若要嚴格資安，需要後端登入。

上線流程

從本機到公開網址

  1. 建立檔案HTML、CSS、JavaScript、GAS 程式碼。
  2. Git 版本記錄初始化、提交、推送。
  3. GitHub Repository建立 `satir-five-freedoms-survey`。
  4. GitHub Pages啟用 main branch 根目錄。
  5. Google Drive 備份複製到 `GPTcodex資料夾/Satir`。

協作心法

提問越像「教練回饋」，AI 越能完成專案

先說目的這是給誰用？在什麼場景用？
再給資料題目、欄位、分類、顏色、角色。
看圖回饋指出哪裡太擠、看不到、順序不對。
逐步加功能先原型，再資料，再權限，再上線。
要求驗證請 AI 檢查語法、推上 GitHub、確認 Pages 狀態。
保留版本GitHub 與 Google Drive 各保留一份。

  5. 給同事模板讓他們帶自己的題目試做。
